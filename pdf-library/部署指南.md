# PDF 图书馆 - 部署指南

## 概述

PDF 图书馆是一个基于 Vue 3 + Vite 的 PDF 图书分享平台前端应用。

## 技术栈

- **前端框架**: Vue 3
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router
- **HTTP 客户端**: Axios
- **字体**: Cormorant Garamond + Libre Baskerville（编辑风格）

## 项目结构

```
pdf-library/
├── src/
│   ├── api/              # API 接口封装
│   ├── components/       # 公共组件
│   ├── router/           # 路由配置
│   ├── stores/           # Pinia 状态管理
│   ├── views/            # 页面组件
│   │   ├── home/         # 首页
│   │   ├── auth/        # 登录/注册
│   │   ├── book/        # 图书列表、详情、上传、搜索
│   │   ├── user/        # 用户中心、收藏、积分、通知
│   │   └── admin/       # 管理后台
│   ├── App.vue           # 根组件
│   ├── main.js           # 入口文件
│   └── style.css         # 全局样式
├── public/               # 静态资源
├── index.html            # HTML 入口
├── package.json          # 项目配置
├── vite.config.js        # Vite 配置
└── vercel.json           # Vercel 部署配置
```

## 本地开发

### 1. 安装依赖

```bash
cd pdf-library
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:5173

### 3. 构建生产版本

```bash
npm run build
```

构建产物在 `dist/` 目录。

## 部署到 Vercel

### 方式一：Vercel CLI（推荐）

```bash
# 1. 安装 Vercel CLI
npm i -g vercel

# 2. 进入项目目录
cd pdf-library

# 3. 登录 Vercel（首次需要）
vercel login

# 4. 部署
vercel
```

按照提示完成部署，Vercel 会自动分配一个域名。

### 方式二：GitHub 自动部署

#### 步骤 1：推送代码到 GitHub

```bash
# 初始化 Git（如果未初始化）
git init

# 添加所有文件
git add .

# 提交
git commit -m "PDF Library 前端项目"

# 创建 master 分支（如果需要）
git branch -M master

# 添加远程仓库
git remote add origin https://github.com/你的用户名/pdf-library.git

# 推送
git push -u origin master
```

#### 步骤 2：在 Vercel 导入项目

1. 访问 https://vercel.com 并登录
2. 点击 "Add New..." → Project
3. 选择 "Import Git Repository"
4. 选择刚推送的 `pdf-library` 仓库
5. 配置部署设置：
   - Framework Preset: `Vite`
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. 点击 "Deploy"

#### 步骤 3：绑定自定义域名（可选）

在 Vercel 项目设置中添加自定义域名。

## API 对接

前端需要配合后端 API 服务，默认请求 `/api` 路径。

在 `vite.config.js` 中配置代理：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:3000',
      changeOrigin: true
    }
  }
}
```

### 主要 API 端点

| 模块 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 认证 | `/api/auth/login` | POST | 登录 |

| 认证 | `/api/auth/logout` | POST | 登出 |
| 用户 | `/api/user/profile` | GET/PUT | 用户资料 |
| 用户 | `/api/user/points` | GET | 积分信息 |
| 图书 | `/api/books` | GET | 图书列表 |
| 图书 | `/api/books/:id` | GET | 图书详情 |
| 图书 | `/api/books` | POST | 上传图书 |
| 图书 | `/api/books/:id/download` | POST | 下载图书 |
| 分类 | `/api/categories` | GET | 分类列表 |
| 标签 | `/api/tags` | GET | 标签列表 |
| 收藏 | `/api/favorites` | GET/POST/DELETE | 收藏操作 |
| 首页 | `/api/home/hot` | GET | 热门图书 |
| 首页 | `/api/home/random` | GET | 随机图书 |
| 通知 | `/api/notifications` | GET | 通知列表 |
| 管理 | `/api/admin/books/pending` | GET | 待审核列表 |
| 管理 | `/api/admin/books/:id/review` | POST | 审核图书 |
| 管理 | `/api/admin/users` | GET | 用户列表 |
| 管理 | `/api/admin/stats` | GET | 统计数据 |

## 注意事项

1. **后端依赖**：前端需要后端 API 才能完整运行
2. **跨域配置**：生产环境需配置 CORS
3. **Session 认证**：使用 Cookie-based Session
4. **文件上传**：PDF 文件限制 50MB
5. **积分系统**：下载图书扣除积分，上传获得积分

## 常见问题

### Q: 部署后页面空白
**A**: 检查 `vercel.json` 配置，确保正确指向 `dist` 目录

### Q: API 请求失败
**A**: 确认后端 API 服务已启动，或配置正确的 API 地址

### Q: 字体加载失败
**A**: Google Fonts 可能被墙，建议使用国内字体服务或本地字体

### Q: 如何修改 API 地址？
**A**: 在 `vite.config.js` 中修改 proxy 配置，或使用环境变量
